# Onboard for Junior Analyst

Привет. Ты это читаешь - тебя подключили к проекту Traits.
Времени на раскачку нет, но чтобы ты не наделал глупостей, вот список core концепций проекта.

## Словарь

1. **UUID (Universally Unique Identifier)**
   - Почему мы переходим на него с обычных ID (1, 2, 3)?
   - Что такое коллизии при мерже баз?

2. **ER-Diagram (Entity-Relationship)**
   - В чём разница связи "Один-ко-Многим" (1:M) и "Многие-ко-Многим" (M:M)?
   - Что такое "Связующая таблица" (Junction table)?

3. **REST API Design**
   - Чем GET отличается от POST помимо очевидного?
   - Query Params и Path Params, разница?
   - Фишки .JSON формата.

4. **OpenAPI (Swagger)**
   - Это стандарт описания API. `.yaml` файлы, все дела. В гугле не блокируем.

## Инструменты
- **Draw.io** — рисовать схемы не от руки
- **Swagger Editor** — писать контракт API.
- **Google Docs** — описывать логику текстом.

3. Чек противоречий
4. Создай решение в папке `/design`.
