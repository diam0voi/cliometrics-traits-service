# BRD: Traits System

**Цель:** Повысить реиграбельность симуляций за счет добавления RPG-элементов лидерам.

## Функциональные требования
1. **Хранение трейтов:**
   - Новый сервис должен быть полностью изолирован от старого монолита.
   - База данных: PostgreSQL.
   - Идентификаторы: **Только UUID**. Мы отказываемся от INT id, так как планируем шардирование базы в будущем.

2. **Сущность "Трейт":**
   - Название (например, "Железный кулак").
   - Описание.
   - Иконка (на CDN).
   - Эффекты (JSON-структ математики "битвы").

3. **Привязка:**
   - Трейты могут быть временными (например, действуют только с 1939 по 1941 год).
   - Один лидер может иметь сколько угодно трейтов.
   - Один и тот же трейт может быть у разных лидеров (например, "Алкоголик" может быть и у Черчилля, и у Ельцина).

4. **API:**
   - Фронтенд должен получать список трейтов для конкретного лидера.
   - Фронтенд должен иметь возможность запросить трейты сразу для списка лидеров (чтобы не ддосить сервер).

## Нефункциональные требования
- Response time < 50ms.
- Документация в формате OpenAPI 3.0.
